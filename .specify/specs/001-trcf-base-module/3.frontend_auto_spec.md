> **Mục tiêu:** Đây là bản thiết kế **CMS Blueprint** — đọc spec này là đủ để dựng lại toàn bộ frontend.

## 5. API Client — `lib/api.ts`

> Mọi request tự gắn `Authorization: Bearer <token>`. 401 → redirect login.

```typescript
const API_URL = process.env.NEXT_PUBLIC_API_URL || "http://localhost:8000";

async function request<T>(path: string, options?: RequestInit): Promise<T> {
  const token = getAccessToken();
  const res = await fetch(`${API_URL}${path}`, {
    headers: {
      "Content-Type": "application/json",
      ...(token ? { Authorization: `Bearer ${token}` } : {}),
    },
    ...options,
  });
  if (res.status === 401) { window.location.href = "/login"; throw new Error("Chưa đăng nhập"); }
  if (!res.ok) { const e = await res.json().catch(() => ({})); throw new Error(e.detail || `HTTP ${res.status}`); }
  return res.json();
}

// Unwrap standard response: { success, data, message } → data
async function apiGet<T>(path: string): Promise<T> {
  const resp = await request<{ success: boolean; data: T }>( path);
  return resp.data;
}

// APIs
export const schemaApi  = { get: (name: string) => apiGet<ModuleSchema>(`/${name}/meta/schema`) };
export const menuApi    = { getModules: () => apiGet<MenuModule[]>("/modules/menu") };
export const settingsApi = {
  getAll: () => apiGet<SystemSetting[]>("/system-settings/"),
  update: (mod: string, key: string, value: string) =>
    request(`/system-settings/${mod}/${key}`, { method: "PUT", body: JSON.stringify({ value }) }),
};

// Generic CRUD factory → tạo API cho bất kì module
export function createApiModule(name: string) {
  return {
    list:    (params) => request(`/${name}/?${new URLSearchParams(params)}`),   // → { items, total }
    get:     (id)     => apiGet(`/${name}/${id}`),
    create:  (data)   => request(`/${name}/`, { method: "POST", body: JSON.stringify(data) }),
    update:  (id, data) => request(`/${name}/${id}`, { method: "PUT", body: JSON.stringify(data) }),
    delete:  (id)     => request(`/${name}/${id}`, { method: "DELETE" }),
    archive: (id)     => request(`/${name}/${id}/archive`, { method: "POST" }),
    restore: (id)     => request(`/${name}/${id}/restore`, { method: "POST" }),
    listAll: ()       => apiGet(`/${name}/?limit=200`).then(d => d.items),      // FK options
  };
}
```

#### Table Column Logic (`_list_columns` Tier System)

| Tier | Condition | Table Columns |
|---|---|---|
| **Luôn ẩn** | `id`, `created_by`, `updated_by`, `version` | ❌ |
| **Override** | `schema.list_columns = ["name", "price"]` | Chỉ hiện cột này, theo thứ tự list |
| **Mặc định** | `schema.list_columns = null` | Tất cả fields (trừ luôn ẩn) |

```typescript
const ALWAYS_HIDDEN = ["created_by", "updated_by", "version"];
function getTableFields(schema): SchemaField[] {
  if (schema.list_columns?.length) {
    return schema.list_columns.map(name => schema.fields.find(f => f.name === name)).filter(Boolean);
  }
  return schema.fields.filter(f => !ALWAYS_HIDDEN.includes(f.name) && f.name !== "id");
}
```

#### Form Field Logic

Form chỉ hiện fields **không phải** readonly và **không phải** system columns:

```typescript
function getFormFields(schema): SchemaField[] {
  return schema.fields.filter(f =>
    !f.readonly && f.name !== "id" && !["created_by", "updated_by", "version"].includes(f.name)
  );
}
```

### 7c. FieldRenderer — Central Dispatcher

`FieldRenderer` là component duy nhất mà `SmartCrudPage` và `AutoSettingsPage` gọi. Nó quyết định component nào sẽ render dựa trên `field.type` và `field.ui_type`.

```typescript
export const FieldMap: Record<string, FieldComponent> = {
  string:     TextField,
  integer:    NumberField,
  decimal:    NumberField,
  boolean:    BooleanField,
  selection:  SelectField,
  foreignkey: ForeignKeyField,
  one2many:   OneToManyField,
  rich_text:  RichTextField,
  color:      ColorField,
};

export function FieldRenderer(props: FieldBaseProps) {
  const { field } = props;

  // Priority 1: ui_type override (rich_text, color luôn ưu tiên)
  if (field.ui_type === "rich_text") return <RichTextField {...props} />;
  if (field.ui_type === "color")     return <ColorField {...props} />;

  // Priority 2: Lookup từ FieldMap theo field.type
  const Component = FieldMap[field.type] || TextField;
  return <Component {...props} />;
}
```

**Quy tắc dispatch:**
1. `ui_type` được kiểm tra trước — cho phép override bất kỳ `type` nào (ví dụ: `type: "string"` + `ui_type: "rich_text"` → render Tiptap editor thay vì Input)
2. Nếu không có `ui_type` đặc biệt → lookup `field.type` trong `FieldMap`
3. Fallback cuối cùng: `TextField` (luôn an toàn)

### 7e. Form State Management — `react-hook-form`

`SmartCrudPage` sử dụng `react-hook-form` để quản lý toàn bộ form state:

```tsx
import { useForm, FormProvider, useFormContext } from "react-hook-form";

// SmartCrudPage:
const methods = useForm({ defaultValues: {} });
return (
  <FormProvider {...methods}>
    {/* ... table, drawer, dialogs ... */}
    <RecordDrawer ... />
  </FormProvider>
);

// RecordDrawer (bên trong FormProvider):
const methods = useFormContext();
<FieldRenderer
  field={field}
  value={methods.watch(field.name)}
  onChange={(v) => methods.setValue(field.name, v, { shouldDirty: true })}
  error={methods.formState.errors[field.name]?.message}
/>

// Submit:
const data = methods.getValues(); // Lấy toàn bộ data bao gồm cả one2many arrays
await api.create(data);           // Gửi 1 request duy nhất cho cả header + lines
```

**Tại sao dùng react-hook-form?**
- `useFieldArray` — quản lý mảng dòng con (one2many) mà không mất performance
- `useFormContext` — cho phép nested components (RecordDrawer, OneToManyField, RowFieldRenderer) truy cập form state mà không cần prop drilling
- `watch()` — reactive value reading (UI auto-update khi field thay đổi)
- `getValues()` — lấy toàn bộ data tree bao gồm nested arrays cho API submit

## 16. Implementation Notes

### 15a. Datetime format trong API responses

Backend serialize datetime sang **ISO 8601 string**: `"2024-03-26T15:30:00"`. Frontend parse: `new Date(value)`.

### 15b. Decimal fields

Được trả về là `number` (float). Frontend format VND: `num.toLocaleString("vi-VN") + " ₫"`.

### 15c. Auto system columns luôn filterable

Ngoài `filter_fields`, các cột `created_at`, `updated_at`, `created_by`, `updated_by` **luôn** hỗ trợ filter param.

### 15d. Settings endpoint filter

`GET /system-settings/` trả **tất cả** settings. Frontend filter client-side: `all.filter(s => s.module_name === moduleName)`.

### 15e. DrawerClose component

Vaul `DrawerClose` dùng `asChild` pattern (không dùng `render` prop):
```tsx
<DrawerClose asChild>
  <Button variant="outline">Huỷ</Button>
</DrawerClose>
```

### 15f. Server-side pagination

Table dùng **server-side pagination** (skip/limit params), không client-side. TanStack table chỉ render records đã fetch, pagination buttons gọi API lại.

## 17. 🔥 Lưu ý Kỹ Thuật cho AI/Speckt (Critical Rebuild Notes)

> **Để rebuild frontend thành công 100%, AI cần tuân thủ các quy tắc "bất di bất dịch" sau:**

1.  **Strict Font/Scale:** Mọi class Tailwind phải được bao bọc bởi `:root { font-size: 125% }`. Không được hardcode `text-base` nếu không dùng `rem`. Font chính là **Geist Sans**. Base font size 125% tương đương 20px.
2.  **Icon Mapping:** `menu_icon` có thể là Emoji hoặc String (ví dụ: "coffee"). Frontend cần một component `IconRenderer` để map: Nếu là string → render `LucideIcon`, nếu là Emoji → render `span`.
3.  **Client-side Validation:** Cấm hardcode validation logic. Phải dùng `SchemaField.required`, `max_length`, `min_value`, `max_value` từ schema để generate dynamic validation (ví dụ dùng `react-hook-form` + `zod` factory). Các Atom field components (`TextField`, `NumberField`, ...) phải tự động xử lý validation hints (red border, error message) qua `FieldBaseProps.error`.
4.  **Image Upload:** Khi field có `type: "image"`, form phải gửi `multipart/form-data` hoặc Base64 tuỳ backend. Mặc định dùng `Input type="file"` với preview cục bộ trước khi upload.
5.  **Z-index Layering:** Drawer (Vaul) dùng cho form Edit/Create. Dialog dùng cho confirm Delete/Archive. Luôn đảm bảo `portal` của shadcn để tránh hiện tượng Drawer đè lên Dialog.
6.  **Settings Navigation:** Việc chuyển đổi giữa các phân hệ cài đặt PHẢI dùng `Link` component với sub-routes `/admin/settings/[module]`. Không dùng `onClick` + setState thuần tuý để giữ được trạng thái URL và cho phép chia sẻ link trực tiếp đến từng phân hệ cài đặt.
7.  **Auto-sync URL Params:** Trang Cài đặt tổng `/admin/settings` phải tự động Redirect (client-side) sang phân hệ đầu tiên ngay khi fetch được danh sách module. Trạng thái của form cài đặt phải được đồng bộ với URL params (`/admin/settings/[module]`).
8.  **Loading States:** Toàn bộ quá trình fetch schema/settings phải có `Skeleton` loader khớp hoàn toàn với Layout thực tế (tránh bị nhảy giao diện - Layout Shift).
9.  **Color Palette:** Bắt buộc sử dụng hệ màu OKLCH cho toàn bộ UI.
10. **TanStack Table Version:** Đảm bảo sử dụng `@tanstack/react-table` v8 như đã ghi rõ.
11. **401 Handling:** Mọi request API trả về 401 Unauthorized phải tự động redirect về `/login` page.
12. **Fields Directory = Single Source of Truth:** Mọi form input PHẢI đi qua `FieldRenderer` (Section 7c). KHÔNG được render `<Input>` trực tiếp trong `SmartCrudPage` hay `AutoSettingsPage`. Khi thêm field type mới: (a) tạo Atom file `.tsx`, (b) đăng ký vào `FieldMap`, (c) xong — toàn bộ hệ thống tự động nhận.
13. **FormProvider Required:** `SmartCrudPage` PHẢI bọc toàn bộ nội dung trong `<FormProvider {...methods}>` để `OneToManyField`/`RowFieldRenderer` có thể truy cập `useFormContext()`.
14. **File Extension .tsx:** Mọi file trong `components/fields/` chứa JSX PHẢI có extension `.tsx` (không phải `.ts`). Next.js/Turbopack sẽ lỗi build nếu file `.ts` chứa JSX.
