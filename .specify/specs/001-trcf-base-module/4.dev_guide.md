# ☕ CoffeeTree Developer Guide — Module Development

> **Nguyên tắc:** "Code once, run everywhere". Bạn chỉ cần khai báo Business Logic ở backend, hệ thống tự động sinh ra API và Giao diện Admin chuyên nghiệp.

---

## 🚀 Thêm module mới — 3 bước thần tốc

### Bước 1 — Tạo file module
Tạo file `backend/modules/<tên_module>.py`. Kế thừa `TRCFBaseModule` và khai báo cấu trúc:

```python
from base import fields
from base.module import TRCFBaseModule

class Product(TRCFBaseModule):
    # ── 1. Cấu hình định danh ──────────────────────────────────────────
    _name        = "products"       # URL & Table name
    _description = "Thực đơn"       # Tên hiển thị trên trang
    
    # ── 2. Cấu hình Menu Sidebar ──────────────────────────────────────
    _menu_label    = "Món ăn"       # Nhãn menu
    _menu_icon     = "☕"            # Emoji hoặc Lucide icon name
    _menu_parent   = "Bán hàng"     # Tên nhóm menu (hiển thị trực tiếp)
    _menu_sequence = 10             # Thứ tự (càng nhỏ càng lên trên)

    # ── 3. Định nghĩa Fields ──────────────────────────────────────────
    name  = fields.CharField(label="Tên món", required=True)
    price = fields.DecimalField(label="Giá bán", precision=12, scale=0, ui_type="currency")
    is_active = fields.BooleanField(label="Đang bán", default=True)
    category_id = fields.ForeignKeyField(to="categories", label="Danh mục")

    # ── 4. Cấu hình UI nâng cao ───────────────────────────────────────
    _search_fields = ["name"]       # Tìm kiếm theo tên
    _filter_fields = ["category_id"] # Lọc theo danh mục
    _list_columns  = ["name", "price", "is_active"] # Cột hiện trên bảng
```

### Bước 2 — Đăng ký Module
Mở `backend/app/main.py`, thực hiện 3 việc sau:
1. **Import** class module từ file vừa tạo.
2. **Router**: Gọi `app.include_router(ModuleClass.router())`.
3. **Registry**: Thêm class vào list `REGISTERED_MODULES`.

```python
# backend/app/main.py
from modules.invoices import Tax  # 1. Import

app.include_router(Tax.router())   # 2. Router

REGISTERED_MODULES = [
    ...,
    Tax,                           # 3. Thêm vào đây
]
```

### Bước 3 — Khởi tạo Database
Mở Terminal, truy cập vào thư mục `backend` và chạy lệnh sau để tạo bảng trong SQLite (thay `taxes` bằng `_name` của module bạn):

```bash
python app/manage.py reset-module taxes
```
*(Nếu hệ thống báo "Success", module sẽ lập tức xuất hiện trên giao diện Admin và Sidebar).*

## 🛠️ Danh sách Field Types (Cập nhật)

| Field | Tham số nổi bật | UI Rendering |
|---|---|---|
| `CharField` | `max_length`, `unique`, `ui_type="email\|phone"` | Input text |
| `TextField` | `ui_type="textarea"` | Textarea (tự giãn) |
| `RichTextField`| | Trình soạn thảo văn bản (Tiptap) |
| `ColorField` | | Bảng chọn màu (Color Picker) |
| `DecimalField` | `precision=12, scale=2, ui_type="currency"` | Số định dạng tiền tệ (25.000 ₫) |
| `IntField` | `min_value`, `max_value` | Input number |
| `BooleanField` | `default=True` | Switch hoặc Badge (✅/❌) |
| `SelectionField`| `options=["draft", "done"]` | Dropdown Select |
| `ForeignKeyField`| `to="categories", display_field="name"` | Searchable Select (auto-fetch) |
| `OneToManyField`| `to="bom_lines", related_field="bom_id"` | Bảng nhập liệu con (Excel-style) |
| `JSONField` | `default={}` | Input lưu cấu trúc JSON |
| `ImageField` | | File upload + Preview |

## 💡 Mẹo & Lưu ý quan trọng

1. **_name vs Tên file**: `_name` trong module (ví dụ: `"taxes"`) chính là tên bạn dùng trong các lệnh `manage.py` và là đường dẫn API. Nó không nhất thiết phải trùng tên file `.py`.
2. **Failed to fetch**: Nếu giao diện hiện lỗi này, hãy kiểm tra terminal backend. 90% nguyên nhân là do bạn vừa thêm Field mới nhưng chưa chạy lệnh `reset-module` để cập nhật bảng trong Database.
3. **Hot Reload**: Backend chạy với `--reload` sẽ tự nhận diện code mới, nhưng không tự tạo cột mới trong DB. Mỗi khi thêm Field, hãy chạy lại lệnh ở Bước 3.

> **Audit Trail:** Mọi model mặc định có `id`, `created_at`, `updated_at`, `created_by`, `updated_by`, `active`. Bạn không cần khai báo lại.

## ⚡ Nâng cao: Metadata & Hooks

### Cấu hình Table Columns (`_list_columns`)
Kiểm soát thứ tự và các cột hiển thị trên bảng:
```python
_list_columns = ["code", "name", "category_id", "price"]
# Nếu để None: Hệ thống tự hiện tất cả trừ các cột hệ thống.
```

### Computed Fields (Trường tự tính)
Tạo trường dựa trên các trường khác (ví dụ: Thành tiền = Đơn giá * Số lượng):
```python
from base.fields import ComputedField

subtotal = ComputedField(
    fn=lambda r: r.price * r.qty,
    depends=["price", "qty"],
    store=True, # Lưu vào DB để có thể filter/sort
    label="Thành tiền"
)
```

### Business Logic Hooks
Xử lý logic phức tạp trước/sau khi lưu dữ liệu:
```python
async def before_create(self, data: dict, db) -> dict:
    if data["price"] < 0:
        raise ValueError("Giá không được âm")
    return data

async def after_create(self, record, db) -> None:
    # Gửi thông báo, log history...
    pass
```

## ⚙️ Module Settings

Nếu module cần trang cấu hình riêng (như VAT mặc định, Tên cửa hàng...):

```python
_has_settings = True
_settings = [
    {"key": "tax_rate", "label": "Thuế suất (%)", "value_type": "decimal", "default": "10"},
]
```
Truy cập tất cả settings tại Sidebar → **Cài đặt** (trang tập trung cho mọi module).

## 🛡️ Phân quyền (Permission)

Mọi module mặc định yêu cầu đăng nhập. Việc phân quyền chi tiết (Ai được Xem, Sửa, Xóa...) được thực hiện tại:
**Hệ thống → Phân quyền**. Bạn không cần viết code check quyền trong module.
